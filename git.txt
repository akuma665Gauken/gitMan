1. Указываем Имя и Емайл (т.к. каждый коммит будет содержать эти данные)

git config --global user.name "Ishimura Gauken"
git config --global user.email "mogushin87@gmail.com"

опция --global - настройки будут использоваться для всего в git

2. Посмотреть созданные настройки
git config --list

3. Помощь
git help комманда
Н-р: git help commit


4. Добавление ((инициализация) существующего репозитория в git

1) Зайти в нужную папку 
2) git init  (появится скрытая папка .git)

5. Если в папке находились файлы, то они все еще Неотслеживаемые,
их надо закомитить

посмотреть статус файлов 
git status

********************************************************
П-р:
На ветке master

Начальный коммит

Неотслеживаемые файлы:
  (используйте «git add <файл>…», чтобы добавить в то, что будет включено в коммит)

	2.html
	mypod.png

ничего не добавлено в коммит, но есть неотслеживаемые файлы (используйте «git add», чтобы отслеживать их)
**********************************************************

6. Добавить все файлы
git add .

проверить статус 
git status

7. Если внести изменения в существующие файлы и проверить статус
git status

то покажут непроиндексированные изменения

***********************************************
На ветке master

Начальный коммит

Изменения, которые будут включены в коммит:
  (используйте «git rm --cached <файл>…», чтобы убрать из индекса)

	новый файл:    2.html
	новый файл:    mypod.png

Изменения, которые не в индексе для коммита:
  (используйте «git add <файл>…», чтобы добавить файл в индекс)
  (используйте «git checkout -- <файл>…», чтобы отменить изменения
   в рабочем каталоге)

	изменено:      2.html
**************************************************

что бы проиндексировать нужно
git add файл

8. Сделать слепок состояния (комит) файлов:
сначала проиндексировать
git add .

затем написать комит с каким-либо комментарием
git commit -m "first commit"

Проверяем статус
git status

***************************************************
На ветке master
нечего коммитить, нет изменений в рабочем каталоге
[akuma@localhost git1]$ git status 
На ветке master
нечего коммитить, нет изменений в рабочем каталоге
****************************************************

Все нормально

9. Если изменить файлы и проверить статус
git status

то скажут какие файлы изменены

Затем индексируем измененный файл
git add .

и сделать слепок текущего изменения
git commit -m "second commit"


10. Заливка на github
Создать акаунт на github.com

Создать репозиторий на github
например site3

теперь определить удаленный репозиторий (скопировать с сайта)
git remote add origin https://github.com/ishimuraGauken/site3.git
git push -u origin master

Ввести логин и пароль с гитхаба

Мы залили наш проект на Гитхаб в созданный репозиторий site3

11. Склонировать чей либо (или свой) репозиторий с Гитхаба]

Клонирование по HTTPS

git clone https://github.com/ishimuraGauken/site3.git
